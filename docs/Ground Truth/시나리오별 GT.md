## Stage A: 첫 미팅

### A-1. 보험사 AICC 도입 초기 미팅 [`A_보험사_AICC탐색`]
- **상황**: 한화손해보험 초기 탐색 미팅. STT 미도입 상태.
- **핵심 질문 정답**:
    - **Q1. 금융권 레퍼런스?**: KB금융그룹 FCC(7개 계열사 STT), KB손보 PoC, 우리카드 콜봇, LG유플러스 4,500석 센터 운영 사례.
    - **Q2. STT 인식률?**: 기본 80-90%, 금융 특화 학습 후 약 95% (KB손보 기준 상담사 98%, 고객 92%).
    - **Q3. 인프라 변경?**: 게이트웨이 추가 외 대규모 변경 불필요. 녹취 연동 가능.
- **함정 대응**: "금융권 LLM 콜봇 상용 사례는 아직 없다"고 솔직히 인정해야 신뢰 확보.

### A-2. 증권사 콜봇 도입 탐색 [`A_증권사_콜봇탐색`]
- **상황**: 한국투자증권 미팅 기반. BPO 외주 15명 운영 중.
- **핵심 질문 정답**:
    - **Q1. 완전 무인화?**: 불가. 단순 반복 문의 흡수가 목표.
    - **Q2. 완결률?**: 60~80% 수준 설명 (유플러스 사례 연간 800만 건 해결).
    - **Q3. 비용?**: 구독형(채널당 월 과금) vs 구축형(수억 단위, 6~12개월) 비교.
- **함정 대응**: KT와의 기술 차별점(자체 콜센터 운영 노하우) 강조.

### A-3. 상담 어드바이저(KMS/RAG) 상품 소개 [`A_어드바이저_상품소개`]
- **상황**: 우리금융캐피탈·FIS 미팅 기반. 챗봇+채팅 통합 니즈.
- **핵심 질문 정답**:
    - **Q1. 통합 솔루션?**: DELMA 플랫폼 (어드바이저+챗봇+에이전트 빌더 통합).
    - **Q2. 연동?**: 기존 상담 AP(엑스프레임) 임베드 또는 API 연동 가능.
    - **Q3. 효과 측정?**: 콜당 톡타임 30초 감소, 수작업 19초 감소 등 수치 제시.

### A-4. 물류사 AICC 도입 첫 방문 [`A_물류사_AICC_첫방문`]
- **상황**: 한진택배. Genesys Cloud 사용 중.
- **핵심 질문 정답**:
    - **Q1. 콜봇 효율?**: 배송 조회/분실 신고 등 단순 문의 80% 처리 적합.
    - **Q2. Genesys 안정성?**: MSA 기반 모듈화로 범용 CTI 연동 안정성 레퍼런스 보유.
- **함정 대응**: 택배사 특유의 숫자(송장번호) 인식 정확도 기대치 조정 필요.

## Stage B: 솔루션 시연

### B-1. 솔루션 시연 + 사업 효과 설명 [`B_증권사_솔루션시연`]
- **상황**: 대신증권 시연 미팅 기반. 지표(97% 인식률) 시연 후 질문.
- **핵심 질문 정답**:
    - **Q1. 자동 추천?**: 대화 맥락 파악하여 실시간 자동 추천(상담사 선택 방식).
    - **Q2. 레이턴시?**: 클라우드 연동 약 2.5초, 온프레미스 거의 실시간. 중간 멘트/캐싱 활용.
    - **Q3. 성공 요인**: 어드바이저 톡타임 절감(17%) + 콜봇 콜 흡수(16%).

### B-2. 캐피탈사 AICC 기술 Q&A 시연 [`B_캐피탈_기술QnA`]
- **상황**: 금융캐피탈 기술 미팅. 보안 및 연동 중심.
- **핵심 질문 정답**:
    - **Q1. 권한 분리?**: 부서별 저장소 분리 및 RBAC 권한 제어 지원.
    - **Q2. 마스킹?**: NER 적용하여 개인정보 실시간 자동 마스킹.
    - **Q3. 모바일**: 웹 기반 솔루션으로 태블릿/모바일 접근 가능.

### B-3. 도시가스/유틸리티사 AICC 시연 [`B_유틸리티_AICC시연`]
- **상황**: 도시가스사 미팅. 개인화 및 다국어 지원.
- **핵심 질문 정답**:
    - **Q1. 개인화**: 고객 프로필 기반 메뉴/이력 자동 조회 기능.
    - **Q2. 다국어**: 한국어 특화 STT + 외산 STT 하이브리드 지원 설명.
    - **Q3. 재학습**: AI 인더루프를 통해 일 단위 자동 재학습 체계 구축.

## Stage C: 기술 심층 Q&A

### C-1. 대형 금융 STT 기능 요건 상세 논의 [`C_STT_요건정의`]
- **정답**: 실시간 파셜 지원(RT 게이트웨이), DB 암호화(AES-256), 학습-배포 CICD 자동화 파이프라인.

### C-2. 클라우드 콜봇 연동 아키텍처 설계 [`C_콜봇_아키텍처`]
- **정답**: 시스코 PBX 연동용 VBG 서버 필수(16코어), 개인정보 필터링(VBG단), FAQ(룰)+RAG(LLM) 하이브리드 방식.

### C-3. 금융보험사 IT 아키텍처 심층 Q&A [`C_금융보험_IT아키텍처`]
- **정답**: 기존 STT 교체 용이(게이트웨이 방식), Vector DB(Elastic/Mongo) 선택 가이드, 생성형 AI 제외 구축 지원(모듈형).

## Stage D: 견적/제안서 협의

### D-1. STT/TTS 기술 견적 협의 [`D_STT_TTS_견적`]
- **정답**: 채널당 40~50만 원 수준 조율 가능, TTS CPU(24코어) 운영으로 인프라 비용 절감, 리얼타임 API 버전 권장.

### D-2. 멀티벤더 솔루션 비교 협의 [`D_멀티벤더_비교`]
- **정답**: CTI 교체 시 생태계 전체 전환 필요(TCO 관점), PDS 연결률 개선 가치 강조(3.5->5.8만 원 정당화).

### D-3. 구독형 vs 구축형 비용 비교 협의 [`D_구독형_구축형_비용`]
- **정답**: 10억 중반대 시 단계적 구축(1단계 어드바이저) 제안, 표준 API를 통한 벤더 락인 해소 방안.

## Stage E: PoC 사전 협의

### E-1. 아웃바운드 AI 콜봇 PoC 검토 [`E_아웃바운드_콜봇_PoC`]
- **정답**: PoC 기간 최소 2개월(튜너 투입), 1~2주 내에는 구독형 데모 시연으로 대응.

### E-2. 보이스봇 요건정의 / RFP 지원 [`E_보이스봇_요건정의`]
- **정답**: 장비 점검 안내 시나리오(IVR 전진 배치), 재발신 로직(3회 시도), 텍스트 요약(고장 증상 구조화).

### E-3. 불완전판매 모니터링 STT+TA PoC 사전 협의 [`E_불완전판매_STT_PoC`]
- **정답**: 일 5만 건 분산 처리(5~6대 서버), CER 기준 통일 합의, H200 대안(L40S 등) 제시.
