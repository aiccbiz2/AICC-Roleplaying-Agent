### A-1. 보험사 AICC 도입 초기 미팅

| 항목 | 내용 |
|------|------|
| **출처** | 한화손해보험 |
| **데이터 소스** | 회의록 직접 인용 (한화손해보험 AICC도입 미팅 060323, 67분) |

**Q1. 금융권 구축 레퍼런스가 있나요?**

| 핵심 포인트 | 정답 |
|------------|------|
| KB금융그룹 FCC | 7개 계열사(KB손해보험, KB생명 등)에 자체 STT 도입 중. 기존 엔솔루 STT를 LG유플러스 자체 STT로 대체 진행 |
| KB손해보험 PoC | RX/TX 각 100건(총 200건)으로 PoC 수행. 180건 학습, 20건 검증 방식 |
| 우리카드 | 콜봇 구축 완료 (마인드웨어 솔루션, SI 형태) |
| 신한카드 | 오토QA 솔루션 구축 (복스 솔루션 활용), 800~1,500석 규모 |
| LG전자, 쿠쿠 | 2025년부터 AICC 구축 진행 중 |
| 자사(LG유플러스) | 4,500명 규모 콜센터에서 자체 AICC 운영, 2021년부터 구축 |
| LLM 콜봇 사례 | **"금융권에 LLM 기반 콜봇 상용 사례는 아직 없다"고 솔직 인정**. 유플러스 자체 5,000석 콜센터도 LLM 콜봇 미도입 (NLP/시나리오 기반 운영 중) |

**Q2. STT 인식률은 어느 정도인가? (금융 용어 학습)**

| 핵심 포인트 | 정답 |
|------------|------|
| 기본 모델(텔코) | 80~90% (금융 도메인 학습 전) |
| 금융 특화 학습 후 | **약 95%** (KB손해보험 PoC 기준: 상담원 98%, 고객 92%) |
| 금융 모델 범용성 | KB 종속 아닌 **금융권 전반에 범용 사용 가능** |
| 학습 도구 | 웹 형태 사전 관리 + 파일 업로드 기능 제공. 금융 용어/상품명 파일 업로드 후 클릭 적용 가능 |
| 인식률 측정 | 파일 단위 인식률 측정 기능 웹 제공 (정답지 대비 비교). 대량 데이터는 서버 레벨 처리 |
| 표준 인식률 범위 | 일반적으로 **93~96%** 수준 |

**Q3. 레거시 인프라 변경 없이 STT 도입이 가능한가?**

| 핵심 포인트 | 정답 |
|------------|------|
| 인프라 변경 | **인프라 구성의 변경은 불필요**. 게이트웨이 정도만 추가 |
| 하드웨어 | 일부 하드웨어 스펙 추가 가능성 있으나 대규모 변경 없음 |
| 녹취 연동 | 기존 동방녹취에서 RX/TX 분리된 음원 파일로 STT 연동 가능 |
| 실시간 미러링 | 녹취 쪽에서 실시간 음원 미러링 인터페이스 필요 가능 (녹취 업체와 협의) |
| 구독형 제공 | 구축형 외에 **구독형으로도 제공 가능** |
| 구축 일정 | STT/TTS만 적용 시 최소 **4개월**, 음성봇 전체 구축 시 **6~8개월** (업무 범위에 따라 최대 1년) |

**Q4. 센터별(인바운드/TM) 각각 구축해야 하나?**

| 핵심 포인트 | 정답 |
|------------|------|
| 큰 시스템 | 비슷하게 구성 가능 |
| 세부 차이 | 인바운드(고객센터): 지식 추천/고객 의도 파악 / TM(아웃바운드): 추천 상품, 대본, 후처리가 다름 |
| STT 공유 | 동일 STT 엔진 사용 가능 |
| 사이트 라이센스 | CS, TM, 보상 등 여러 업무에서 사용 가능 (단, 유저 수는 정해야 함) |
| 모바일 녹취 | 유선 녹취는 바로 가능. 모바일 녹취 STT는 별도 검토 필요 |

**Q5. KMS 전담자가 필요한가?**

| 핵심 포인트 | 정답 |
|------------|------|
| 초기 세팅 | KMS 데이터를 LLM이 읽을 수 있게 정제하는 과정은 기술 조직에서 수행 |
| 운영 단계 | **일 단위 배치**로 자동 업데이트. 현업 부서가 수시 업데이트하면 센터에서 가져가는 구조 |
| 유플러스 사례 | 초기 **8명** 투입 -> AI 인더루프 자동화 후 **전담자 불필요**. 수작업 5,760분 -> 40분 단축 |
| 신규 시나리오 | 현업에서 직접 추가 어려움 -> IT와 짝으로 투입 권장. 계약 시 **연간 무상 추가 건수** 합의 필요 |

**함정 대응 가이드**
- **금융권 LLM 콜봇 사례 없음**: "솔직히 금융권에서 LLM 콜봇 상용화 사례는 아직 없습니다. 유플러스 자체 콜센터도 NLP 기반이고, LLM 콜봇은 5월 오픈 예정입니다. LLM은 물어볼 때마다 다른 답을 하는 이슈가 있어 금융권은 시나리오 기반이 현실적입니다." -- 솔직한 인정이 신뢰 구축의 핵심
- **녹취 데이터 확보 방안**: "내부 데이터 외부 반출에 보안 이슈가 있다면, KB손보 사례처럼 RX/TX 각 100건의 음원 파일을 제공받아 PoC 진행 가능. 데이터 증강 기술로 소량 데이터에서도 유의미한 결과 도출합니다."

### A-2. 증권사 콜봇 도입 탐색

| 항목 | 내용 |
|------|------|
| **출처** | 한국투자증권 |
| **데이터 소스** | 패턴DB 활용 + 대신증권 회의록 참조 (동일 금융/증권 도메인) |

**Q1. 콜봇 완전 무인화가 가능한가?**

| 핵심 포인트 | 정답 |
|------------|------|
| 결론 | **완전 무인화 불가**. AI를 싫어하는 고객, 돈/계좌 관련 크리티컬 문의는 반드시 상담사 연결 필요 |
| 현실적 목표 | 단순 반복 문의(시세, 영업시간, 잔고)를 콜봇이 흡수 -> 상담사 연결 콜 자체를 줄이는 것 |
| 대신증권 참조 | "보이스봇은 지금 도입해도 솔직히 기대하기 쉽지 않다." 주문까지 가능한 시점은 2~3년 소요 |
| 금융 리스크 | 오답변 시 금융 사고 직결 -- 금융권이 콜봇/LLM을 가장 꺼리는 이유 1순위 |
| 권장 접근 | 내부 직원용(어드바이저, KMS) 1년 운영 -> 검증 후 대고객 서비스 확대 |

**Q2. 콜봇 완결률은 보통 얼마나 되나?**

| 핵심 포인트 | 정답 |
|------------|------|
| 유플러스 사례 | 룰 베이스 콜봇 기준 연간 **800만 건** 고객 문제 해결 |
| LLM 챗봇 효과 | 대화 건수 약 **60% 감소**, NPS **50포인트** 상승 |
| 핵심 변수 | 업무 복잡도/도메인에 따라 상이. "콜봇이 처리할 업무"를 먼저 정의해야 완결률 논의 가능 |
| 룰 베이스 장점 | 정형화된 업무는 완결률 높음. 실수 없고, 할루시네이션 위험 없음 |

**Q3. 구독형/구축형 비용은?**

| 핵심 포인트 | 정답 |
|------------|------|
| 구독형 | 채널당 월 과금 방식. 소규모 시작 가능. BPO 10~15명 규모라면 초기 부담 적음 |
| 구축형 | 수억 단위 (업무 범위에 따라 변동). STT+콜봇+어드바이저 풀 구축 시 6~12개월 |
| 채널 구조 | STT 상담원 1명당 **2채널** 필요 (RX/TX 분리) |
| 전환 | 구독형 테스트 후 구축형 전환 가능. 아키텍처 차이 있으므로 일부 재구축 요소 발생 |

**Q4. KT 대비 기술 수준 차이는?**

| 핵심 포인트 | 정답 |
|------------|------|
| 원칙 | **경쟁사를 근거 없이 폄하하면 역효과**. 객관적 사실 위주 답변 |
| 차별점 | 자사 5,000석 콜센터에서 직접 운영하며 검증한 솔루션. 운영 노하우 + 솔루션 개발 동시 보유 |
| 원벤더 장점 | STT, 어드바이저, QA, 콜봇 모두 자체 개발 -> "서로 책임 떠넘기기" 문제 없음 |
| 권장 표현 | "저희 솔루션의 검증된 성과"를 수치로 제시하는 것이 효과적 |

**Q5. BPO 외주 10~15명 대체 가능한가?**

| 핵심 포인트 | 정답 |
|------------|------|
| 결론 | 완전 대체 불가. 단순 문의 콜봇 흡수 + 업무 효율화로 **인원 최적화** 가능 |
| 유플러스 사례 | 2년에 걸쳐 **507명 자연 감소** (인위적 해고 아닌 퇴사자 미충원) |
| 접근법 | 콜 데이터 분석 -> 콜봇 처리 가능 업무 식별 -> 단계적 적용 |
| 효과 시점 | 구축 후 바로 효과가 아닌, 데일리 피드백 루프 통한 지속 개선 필요 (최소 6개월~1년) |

**함정 대응 가이드**
- **경쟁사(KT) 비교**: "KT도 좋은 솔루션을 보유하고 있습니다. 다만 저희는 자체 콜센터에서 2년간 운영하며 검증한 피드백 루프와 운영 노하우가 차별점입니다." -- 근거 없는 우위 주장 금지
- **완전 무인화 기대치 조정**: "무인화보다 '스마트화'가 맞는 표현입니다. AI가 단순 업무를 흡수하고, 상담사는 복잡한 고가치 상담에 집중하는 구조입니다."

### A-3. 상담 어드바이저(KMS/RAG) 상품 소개

| 항목 | 내용 |
|------|------|
| **출처** | 우리금융캐피탈 / FIS |
| **데이터 소스** | 회의록 직접 인용 (우리금융캐피탈 다이퀘스트 상품소개 QnA 260311, 48분) |

**Q1. 챗봇+채팅 통합 솔루션이 있나?**

| 핵심 포인트 | 정답 |
|------------|------|
| 통합 제공 | 어드바이저(상담 지원) + 챗봇(대고객) + 에이전트 빌더를 **DELMA 플랫폼**에서 통합 제공 |
| 워크플로우 | 에이전트 빌더에서 질문 분류기, 검색, 응답 생성 등을 드래그&드롭으로 구성 가능 |
| 모델 선택 | 대화창에서 모델 선택하여 답변 비교 가능. 라마, QWen, DPT 등 다양한 모델 탑재 |
| 멀티턴 | 문맥 유지 대화 지원. 앞선 대화 맥락 기반 답변 제공 |

**Q2. 어드바이저는 별도 구매인가, 패키지인가?**

| 핵심 포인트 | 정답 |
|------------|------|
| 패키지 구성 | "디 플랫폼"이라는 이름 안에 어드바이저, 상품 추천(디필릭), ML옵스, LLM옵스를 **통합 제공** |
| 하드웨어 | 딥러닝 상품 추천 에이전트는 별도 GPU 서버, 플랫폼은 DELMA로 일괄 견적 |
| 분리 견적 | 요청 시 솔루션별 분리 견적 제공 가능 |
| STT 별도 | 기존 콜봇 도입 시 납품된 STT 라이선스 잔여분 활용, 연동 비용만 추가 |

**Q3. VDI 없이도 사용 가능한가?**

| 핵심 포인트 | 정답 |
|------------|------|
| VDI 불필요 | **웹 기반 솔루션이므로 VDI 없이 사용 가능** |
| 모바일 지원 | 모바일/태블릿에서 웹 기반 접근 가능. 외부 협력 직원 핸드폰 사용 가능 |
| 구현 방식 | 홈페이지 WAS에 컨테이너 추가 또는 별도 사이트 구성 |

**Q4. 기존 상담 AP(엑스프레임)와 연동 가능한가?**

| 핵심 포인트 | 정답 |
|------------|------|
| 연동 방식 | 엑스프레임 안에 iframe보다 **별도 창으로 띄우는 방식** 권장. API 형태 제공 가능 |
| 콜 인입 연동 | 소프트폰 전화 이벤트 -> URL 호출로 미니 어드바이저 자동 팝업 |
| 기관계 연동 | 고객번호/대출번호 인자값 전달 -> 기관계 데이터 자동 조회/노출 가능 |
| 부서별 차별화 | 대출 부서=대출 이력, 렌터카 부서=차량 정보 등 **부서별 다른 데이터 자동 세팅** |
| 후처리 연동 | 상담 코드 추천, 요약 등 후처리 데이터를 API로 기관계 DB에 적재 가능 |

**Q5. AI 도입 효과를 어떻게 측정하나?**

| 핵심 포인트 | 정답 |
|------------|------|
| 유플러스 수치 | 콜당 톡타임 **30초** 감소, 수작업 콜당 **19초** 감소, 어드바이저 도입 후 콜타임 **20%** 감소 |
| 정량 지표 | 콜타임 변화, 코드 분류 정확도(99%), 요약 정확도(95%), 오토QA 23가지 항목 |
| 피드백 시스템 | 좋아요/싫어요 마크 기능으로 답변 만족도 측정 및 자동 학습 반영 |
| 대시보드 | 실시간 운영 현황 대시보드, 스코어링 대시보드 제공 |

**함정 대응 가이드**
- **기관계(엑스프레임) 연동**: "API 형태로 제공하며, 기존 상담 AP 안에 임베드하거나 별도 창 모두 가능합니다. 콜 인입 시 고객번호를 파라미터로 던지면 기관계 정보를 자동 조회하여 노출합니다. 부서별(대출, 렌터카, 해피콜) 다른 데이터 세팅도 가능합니다."

### A-4. 물류사 AICC 도입 첫 방문

| 도메인 학습 | 도시가스 고유 용어(열량 단위, 배관 용어)는 사전 등록 및 도메인 특화 학습으로 인식률 개선 |
| 소음 필터링 | STT에 소음 필터링 기능 -> 외부 통화 시에도 정확도 유지 |

**Q3. LLM 토큰 비용은?**

| 핵심 포인트 | 정답 |
|------------|------|
| 온프레미스 권장 | 온프레미스(SLM) 기반이면 **토큰 과금 없음**. GPU 장비 초기 투자만 |
| SLM 성능 | QWen 32B, GPT OSS 120B 모델로 **GPT-4.0 수준 이상** 성능 (유플러스 검증) |
| 하이브리드 전략 | 단순 분류/요약은 경량 모델(BERT, 익시젠 2.4B), 복잡한 RAG는 대형 LLM -> **용도별 모델 분리**로 비용 최적화 |
| 모델 교체 | DELMA 플랫폼에서 모델 선택/교체 가능 -> 모델 종속 리스크 없음 |

**Q4. 보이는 콜봇(Visual IVR)이 가능한가?**

| 핵심 포인트 | 정답 |
|------------|------|
| 운영 중 | 유플러스가 **실제 운영 중**. "콜봇의 멀티모달 + 보이는 시나리오" |
| 구현 방식 | 음성 통화 중 고객에게 화면 동시 제공. URL 발송 -> 화면에서 정보 확인/입력 |
| 활용 사례 | 청구서 실시간 화면 제공, 이메일/주소/계좌번호 화면 입력, 가입 절차 안내 |
| 효과 | 음성만으로 전달 어려운 긴 내용 -> 텍스트+화면으로 전달도 향상, 재인입 감소 |
| 도시가스 적용 | 요금 조회, 검침 일정, 가스 안전 점검 예약 확인 등에 적용 가능 |

**Q5. AI 오인식 시 재학습은?**

| 핵심 포인트 | 정답 |
|------------|------|
| AI 인더루프 | TA 분류 코드와 상담사 실제 선택 코드 **불일치 시 자동 샘플링** |
| 자동 재학습 | LLM 정답 판정 -> 핵심 발화 추출 -> 벡터DB 임베딩까지 자동화. **5,760분->40분** 단축 |
| STT 자동 개선 | STT 데이터 자동 개선 기능 **7월** 적용 예정 |
| 오류 제보 | 리더/평가자가 포털에서 오류 신고 -> 자동 반영 |
| 데일리 피드백 루프 | STT, TTS, 요약, 분류 각 요소 목표 수준 세팅 -> 데일리 모니터링 |
| 학습 도구 | 웹 기반 사전 관리/학습 수행 기능. 파일 업로드 후 클릭 적용 + 인식률 즉시 확인 |

**함정 대응 가이드**
- **멀티 LLM 선택 기준**: "용도별로 최적 모델을 분리합니다. 단순 분류는 BERT, 지식 추천은 RAG+대형 LLM, 코드 추천은 별도 파인튜닝 모델. DELMA 플랫폼에서 모델을 쉽게 교체할 수 있어 모델 종속 리스크가 없습니다."
- **다국어 STT 한국어 특화**: "한국어 모델은 97% 인식률을 보장하고, 타 언어는 IVR에서 언어 선택 후 해당 언어 전용 모델로 분기합니다. 하나의 모델로 처리하면 한국어 인식률이 떨어지므로 분기 처리가 품질 면에서 최적입니다."
