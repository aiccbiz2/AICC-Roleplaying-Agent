### Q1. 지식 추천 기능은 자동인가, 상담사 요청 시 나오는 건가? (대신증권)

| 핵심 포인트 | 정답 |
|------------|------|
| 작동 방식 | **자동 추천**. 고객-상담사 대화 맥락을 자동 파악하여 질문을 먼저 띄워준다 |
| 상담사 차이 대응 | 초급/중급 상담사마다 수준이 다르므로, 질문을 먼저 띄우고 필요하면 선택 |
| 검색 모드 | 퀵 모드(빠른 응답), 띵킹 모드(복잡한 질문, KMS 정확 검색), 오토 모드(자동 판단) 3가지 |
| 수동 검색 | 추천되지 않는 지식이 필요할 때 하단 검색으로 직접 검색 가능 |
| 응답 속도 | **0.2~0.3초 이내** (SLA 기준 0.3초). STT만 보면 0.1초 이내 |
| 정확도 | 내부 기준 **97% 이상** |
| 기술 기반 | sLLM + RAG 기반 솔루션 |

### Q2. 보이스봇 현재 수준은? 시기상조인가? (대신증권)

| 핵심 포인트 | 정답 |
|------------|------|
| 룰 베이스 콜봇 | 지연 시간 이슈 없음 (밀리세컨드 단위 응답). 할루시네이션 없음. 정형화된 업무에 적합 |
| LLM 콜봇 현황 | NH투자증권에서 오픈AI 연동 PoC 진행 중. 2~3번 질문 시 약 **2.5초** 응답 |
| 금융권 현실 | 인바운드 보이스봇 도입 거의 없음. 우리카드 3년 전 도입 → 현재 거의 미사용 |
| 가장 큰 리스크 | 오답/오상담 → **금융사고 리스크** (할루시네이션) |
| 현실적 활용처 | **아웃바운드 콜봇**(고지용, 불완전판매 체크)이 더 현실적 |
| 권장 순서 | 룰 베이스 먼저 적용 → 데이터 축적 → LLM 에이전트 확장 |
| LLM 콜봇 일정 | **2026년 5월 오픈 예정** (PoC 진행 후) |

### Q3. LLM 콜봇 응답속도 2.5초 — 실시간 가능한가? (대신증권)

| 핵심 포인트 | 정답 |
|------------|------|
| 현재 속도 | 클라우드(오픈AI 연동) 기준 약 **2.5초** |
| 온프레미스 | **거의 실시간** 가능 |
| 체감 지연 줄이기 | ① 고객 질의 요약 반복("이런 것을 질문하셨군요") 중간 멘트 삽입 |
| 캐싱 | ② 동일 질문 반복 시 LLM 거치지 않고 캐싱 저장소에서 즉시 응답 |
| 보이스 투 보이스 | ③ STT→LLM→TTS 단계를 거치지 않는 직접 처리 모델 별도 준비 중 (기술 검증 2025.12 완료, 온프레미스 모델 아직 미출시) |
| 비용 이슈 | 보이스 투 보이스 클라우드 방식은 매우 비싸다 (큰 모델 필요). 현실적으로 룰베이스→LLM 텍스트 전환이 우선 |

### Q5. 오토QA가 답변 내용 정확성까지 평가하나? (놀유니버스)

| 핵심 포인트 | 정답 |
|------------|------|
| 현재 수준 | 상담 프로세스(인사, 발음, 습관어, 존칭 등) 평가는 **가능** |
| 내용 정확성 | "정상 상담인가/오답인가" 판단은 **아직 안 됨** — 고도화 중 |
| 고도화 방향 | KMS(상담 가이드) 정보와 상담사 발화 내용 일치 여부를 매 시점마다 체크하는 모델 기획 중 |
| 현재 감지 방식 | "죄송합니다, 제가 잘못 설명드렸는데" 등 비정상 표현을 대화 흐름상 감지 |
| 평가 항목 수 | AI 평가 **19개** 항목, 세분화 시 **24개** |
| 평가 기준 | 한국표준협회, 능률협회 기준 적용 (발음, 속도, 언어습관, 맞이/종료인사, 존칭, 추가 문의 등) |
| 전수 평가 | 기존 인당 월 1콜(0.2%) → 오토QA 이후 **전수 100% 평가** |
| 결과 제공 | 상담 종료 후 **5분 이내** |
| 상담사 거부감 | 점수 직접 노출 지양, "어느 부분이 부족해"라는 **코칭 형태**로 제공 |

### Q7. KMS 전담 인원은 몇 명 필요한가? (놀유니버스)

| 핵심 포인트 | 정답 |
|------------|------|
| KMS 구축 전담 | **2명** |
| 신규 상품/공지/QnA 포함 | **5명** (기존 업무 겸직) |
| 운영 방식 | 기존 상담 업무 공지 담당자가 어드바이저 KMS 구축까지 겸무 |
| 자동화 방향 | **AI 인더루프(AI-in-the-loop)** 자체 제품 준비 중. RAG 기반 지식 정확도 체크 + 통계적 오류 자동 식별 → 사람 작업 최소화. 올해 내 적용 목표 |
| LG U+ 사례 | 초기 8명 투입 → AI 인더루프 자동화 후 전담자 불필요. 수작업 5,760분→40분 단축 |
| 갱신 주기 | **일 배치(daily batch)**로 자동 업데이트 |

### 함정 5: 금융권 어드바이저 납품 없음
