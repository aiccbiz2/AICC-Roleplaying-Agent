### A-1. 보험사 AICC 도입 초기 미팅

**Q1. 금융권 구축 레퍼런스가 있나요?**

| 핵심 포인트 | 정답 |
|------------|------|
| KB금융그룹 FCC | 7개 계열사(KB손해보험, KB생명 등)에 자체 STT 도입 중. 기존 엔솔루 STT를 LG유플러스 자체 STT로 대체 진행 |
| KB손해보험 PoC | RX/TX 각 100건(총 200건)으로 PoC 수행. 180건 학습, 20건 검증 방식 |
| 우리카드 | 콜봇 구축 완료 (마인드웨어 솔루션, SI 형태) |
| 신한카드 | 오토QA 솔루션 구축 (복스 솔루션 활용), 800~1,500석 규모 |
| LG전자, 쿠쿠 | 2025년부터 AICC 구축 진행 중 |
| 자사(LG유플러스) | 4,500명 규모 콜센터에서 자체 AICC 운영, 2021년부터 구축 |
| LLM 콜봇 사례 | **"금융권에 LLM 기반 콜봇 상용 사례는 아직 없다"고 솔직 인정**. 유플러스 자체 5,000석 콜센터도 LLM 콜봇 미도입 (NLP/시나리오 기반 운영 중) |

**Q2. STT 인식률은 어느 정도인가? (금융 용어 학습)**

| 핵심 포인트 | 정답 |
|------------|------|
| 기본 모델(텔코) | 80~90% (금융 도메인 학습 전) |
| 금융 특화 학습 후 | **약 95%** (KB손해보험 PoC 기준: 상담원 98%, 고객 92%) |
| 금융 모델 범용성 | KB 종속 아닌 **금융권 전반에 범용 사용 가능** |
| 학습 도구 | 웹 형태 사전 관리 + 파일 업로드 기능 제공. 금융 용어/상품명 파일 업로드 후 클릭 적용 가능 |
| 인식률 측정 | 파일 단위 인식률 측정 기능 웹 제공 (정답지 대비 비교). 대량 데이터는 서버 레벨 처리 |
| 표준 인식률 범위 | 일반적으로 **93~96%** 수준 |

**Q3. 레거시 인프라 변경 없이 STT 도입이 가능한가?**

| 핵심 포인트 | 정답 |
|------------|------|
| 인프라 변경 | **인프라 구성의 변경은 불필요**. 게이트웨이 정도만 추가 |
| 하드웨어 | 일부 하드웨어 스펙 추가 가능성 있으나 대규모 변경 없음 |
| 녹취 연동 | 기존 동방녹취에서 RX/TX 분리된 음원 파일로 STT 연동 가능 |
| 실시간 미러링 | 녹취 쪽에서 실시간 음원 미러링 인터페이스 필요 가능 (녹취 업체와 협의) |
| 구독형 제공 | 구축형 외에 **구독형으로도 제공 가능** |
| 구축 일정 | STT/TTS만 적용 시 최소 **4개월**, 음성봇 전체 구축 시 **6~8개월** (업무 범위에 따라 최대 1년) |

**Q4. 센터별(인바운드/TM) 각각 구축해야 하나?**

| 핵심 포인트 | 정답 |
|------------|------|
| 큰 시스템 | 비슷하게 구성 가능 |
| 세부 차이 | 인바운드(고객센터): 지식 추천/고객 의도 파악 / TM(아웃바운드): 추천 상품, 대본, 후처리가 다름 |
| STT 공유 | 동일 STT 엔진 사용 가능 |
| 사이트 라이센스 | CS, TM, 보상 등 여러 업무에서 사용 가능 (단, 유저 수는 정해야 함) |
| 모바일 녹취 | 유선 녹취는 바로 가능. 모바일 녹취 STT는 별도 검토 필요 |

**Q5. KMS 전담자가 필요한가?**

| 핵심 포인트 | 정답 |
|------------|------|
| 초기 세팅 | KMS 데이터를 LLM이 읽을 수 있게 정제하는 과정은 기술 조직에서 수행 |
| 운영 단계 | **일 단위 배치**로 자동 업데이트. 현업 부서가 수시 업데이트하면 센터에서 가져가는 구조 |
| 유플러스 사례 | 초기 **8명** 투입 -> AI 인더루프 자동화 후 **전담자 불필요**. 수작업 5,760분 -> 40분 단축 |
| 신규 시나리오 | 현업에서 직접 추가 어려움 -> IT와 짝으로 투입 권장. 계약 시 **연간 무상 추가 건수** 합의 필요 |

**함정 대응 가이드**
- **금융권 LLM 콜봇 사례 없음**: "솔직히 금융권에서 LLM 콜봇 상용화 사례는 아직 없습니다. 유플러스 자체 콜센터도 NLP 기반이고, LLM 콜봇은 5월 오픈 예정입니다."
- **녹취 데이터 확보 방안**: "내부 데이터 외부 반출에 보안 이슈가 있다면, KB손보 사례처럼 RX/TX 각 100건의 음원 파일을 제공받아 PoC 진행 가능."

### A-2. 증권사 콜봇 도입 탐색

| 항목 | 내용 |
|------|------|
| **출처** | 한국투자증권 |
| **데이터 소스** | 패턴DB 활용 + 대신증권 회의록 참조 |

**Q1. 콜봇 완전 무인화가 가능한가?**
- **완전 무인화 불가**. AI를 싫어하는 고객, 돈/계좌 관련 크리티컬 문의는 반드시 상담사 연결 필요.
- **현실적 목표**: 단순 반복 문의를 콜봇이 흡수하여 상담사 연결 콜 자체를 줄이는 것.

**Q2. 콜봇 완결률은 보통 얼마나 되나?**
- **유플러스 사례**: 룰 베이스 콜봇 기준 연간 800만 건 해결.
- **LLM 챗봇 효과**: 대화 건수 약 60% 감소, NPS 50포인트 상승.

**Q3. 구독형/구축형 비용은?**
- **구독형**: 채널당 월 과금 방식. 소규모 시작 가능.
- **구축형**: 수억 단위. STT+콜봇+어드바이저 풀 구축 시 6~12개월.

**Q4. KT 대비 기술 수준 차이는?**
- **차별점**: 자사 5,000석 콜센터에서 직접 운영하며 검증한 솔루션. 운영 노하우 + 솔루션 개발 동시 보유.

**Q5. BPO 외주 10~15명 대체 가능한가?**
- **결론**: 완전 대체 불가. 단순 문의 콜봇 흡수 + 업무 효율화로 인원 최적화 가능.

## Stage B: 솔루션 시연 시나리오 (3개)

### B-1. 솔루션 시연 + 사업 효과 설명

| 항목 | 내용 |
|------|------|
| **출처** | 대신증권 |
| **데이터 소스** | 회의록 직접 인용 (20260325 대신증권 고객시연, 76분) |

**Q1. 지식 추천은 자동인가, 수동(상담사 요청 시) 나오나?**
- **자동 추천**: 고객-상담사 대화 맥락을 파악하여 질문을 자동으로 띄워줌.
- **상담사 역할**: 띄워진 질문이 필요하면 선택(검색 버튼 클릭), 불필요하면 무시.

**Q2. 보이스봇 현재 수준은?**
- **룰 베이스 콜봇**: 현재 운영 중. 지연시간 없음(ms 단위). 할루시네이션 없음.
- **LLM 콜봇**: 개발 중. 2026년 5월 오픈 예정.

**Q3. LLM 콜봇 2.5초 응답이 가능한가?**
- **현재 수준**: 오픈AI 클라우드 연동 시 약 2.5초 피드백.
- **온프레미스**: 클라우드 왕복 없으므로 거의 실시간 가능.
- **대응책**: 중간 멘트 삽입, 캐싱 등으로 체감 지연 감소.

**Q4. 33% 효과의 원인은 무엇인가?**
- **성공 요인**: 어드바이저 톡타임 단축(~17%) + 콜봇 콜 흡수(~16%).
- **인원 절감**: 2년간 507명 자연 감소 (자연 퇴직자 미충원 방식).

## Stage C: 기술 심층 Q&A (3개)

### C-1. 대형 금융 STT 기능 요건 상세 논의

**Q1. 실시간 스트리밍 STT가 기술적으로 가능한가요?**
- **가능**: EC STT 엔진은 실시간 스트리밍 모드 지원. RT 게이트웨이를 통해 구현.

**Q2. STT 결과를 DB에 저장할 수 있어야 합니다.**
- **암호화**: 금융권 필수 요건으로 AES-256 암호화 적용 가능. 보관 기간 차등 설정 가능.

**Q3. 계열사마다 다른 SSO를 전부 연동 가능한가요?**
- **가능**: 다만 계열사 수만큼 개별 연동 작업(SAML, OAuth 등) 필요 (계열사당 약 1~2주).

### C-2. 클라우드 콜봇 연동 아키텍처 설계

**Q1. 중계서버가 꼭 필요한가요?**
- **필요**: 시스코 PBX 연동을 위해 VBG(Voice Bot Gateway) 필수. 사이징은 16코어 권장.

**Q2. FAQ와 RAG 중 어떤 방식으로 처리하나요?**
- **하이브리드**: 단순 문의는 FAQ(룰 베이스), 복합 문의는 RAG(LLM 기반) 하이브리드 접근 권장.

**Q3. 10월 오픈 목표 일정이 가능한가요?**
- **일정**: 음성봇 전체 구축 최소 6개월. STT/TTS만 적용 시 4개월 소요.

## Stage D: 견적/제안서 협의 (3개)

### D-1. STT/TTS 기술 견적 협의

**Q1. 왜 이렇게 비싸냐?**
- **할인**: 리스트 프라이스 대비 실제 입찰 시 대폭 할인 가능. 채널당 40~50만원 수준 조정.

**Q2. GPU 없이 TTS가 가능한가요?**
- **가능**: EC TTS는 CPU(최소 24코어)만으로 운영 가능. 인프라 비용 절감 장점으로 어필.

**Q3. 패키지 형태로 묶어서 제공 가능한가요?**
- **세트 구성**: 콜봇 세트, 대화록 세트 등 용도별 세트 구성 제안 가능.

---

### D-2. 멀티벤더 솔루션 비교 협의

**Q1. CTI가 바뀌면 통계, 소프트폰 전부 바뀌어야 하나요?**
- **연쇄 변경**: CTI 교체 시 통계, 소프트폰, PDS, 녹취 연동 등 생태계 전체 전환 필요 (TCO 관점 접근).

**Q2. P65% 인상되는 PDS 단가 정당화는?**
- **가치**: AI 예측 다이얼링으로 연결률 개선 효과. 실질 필요 채널 수 감소로 TCO는 합리적.

---

### D-3. 구독형 vs 구축형 비용 비교 협의

**Q1. 풀 구축 예산 10억 중반대가 가능한가요?**
- **가이드**: 가능하나 타이트함. 1단계(어드바이저)+2단계(콜봇) 분리 구축 권장.

**Q2. 벤더 종속성(Lock-in) 리스크 대응은?**
- **해법**: 모듈형 아키텍처 + 표준 API + 데이터 소유권 보장 3가지로 해소.
