## 인프라/시스템

| 정식 용어 | 약어 | 교육자료 정의 | 고객 실제 표현 | 비고 |
|-----------|------|-------------|---------------|------|
| Internet Protocol Contact Center | IPCC | IP 기반 차세대 컨택센터. PBX+IVR+CTI+녹취+상담AP로 구성 | "콜센터 인프라", "콜 인프라", "인프라 시스템" | 고객은 IPCC라는 약어보다 "인프라"로 통칭 |

| AI Contact Center | AICC | IPCC에 AI(STT/TTS/LLM) 기술을 접목한 지능형 컨택센터 | "AICC", "AI 콜센터", "AI 상담" | 고객도 AICC 용어 직접 사용 |
| Private Branch Exchange | PBX | 기업 내부 전화 통신을 관리하는 교환기 | "교환기", "전화 시스템" | 1차/2차 교육 핵심 |

| Automatic Response Service | ARS | 단순 음성 안내(일방향) | "ARS" | 고객은 IVR과 구분 없이 사용 |

| Session Border Controller | SBC | IP 방식 프로토콜 변환 + 라우팅 + RTP 스트리밍 장비 | "SBC" | 브리지텍/EICN에서 사용. 어바이어/시스코는 VGW 사용 |
| E1/PRI 회선 | - | 유럽표준 전송방식, 2.048Mbps. ISDN 기반 기업용 전화 회선 | "회선", "E1" | 30채널x회선수=동시통화 가능 수 |
| Session Initiation Protocol | SIP | IP 기반 통신 프로토콜. TDM 대체 | "SIP", "IP 방식" | IP 전환의 핵심 프로토콜 |
| Public Switched Telephone Network | PSTN | 일반 공중 전화망 | "전화망", "통신사 망" | 콜 인입의 시작점 |
| Softphone | - | PC에 설치되는 소프트웨어 전화기 | "소프트폰", "전화 버튼", "받고 끊는 거" | 상담AP에 탑재. 교환기 변경 시 소프트폰도 교체 필요 |
| 상담 어플리케이션 | 상담AP | 상담원이 사용하는 통합 업무 화면 | "상담 화면", "전산", "메인 화면", "계정계 화면" | 고객은 "전산", "엑스프레임" 등 자사 시스템명으로 호칭 |
| 어플라이언스 장비 | - | 벤더 전용 HW+SW 일체형 장비 (어바이어, 시스코) | "전용 장비", "박스" | 범용 서버로 대체 추세 |
| 범용 서버 | - | HP, Dell 등 일반 서버에 SW 설치하는 방식 | "일반 서버", "범용 장비" | 가상화와 결합하여 IPCC 구성 |
| 가상화 | VM | VMware 등 플랫폼 위에 이미지로 시스템 구성 | "VM", "가상화", "이미지로 올린다" | 물리서버 축소, 통합 관리 가능 |
| 하이퍼바이저 | - | 가상화 관리 소프트웨어 (VMware 하이퍼바이저 등) | "하이퍼바이저", "VM웨어" | 통합 서버 플랫폼에서 VM 관리 |

| 소버린 클라우드 | - | 데이터 주권이 보장되는 클라우드 | "소버린 클라우드" | NH투자증권 미팅에서 언급. AWS 사이징과 연관 |
| Microservice Architecture | MSA | 클라우드 기반 마이크로 서비스 아키텍처 | (직접 사용 안 함) | 이상적이나, 국내는 EC2 이미지 올리는 방식이 현실 |

| 스마트안내 | - | 회선 포화 시 고객에게 대기 또는 콜백 안내를 자동 제공하는 IVR 부가 기능 | "스마트안내" | IBK기업은행 미팅 |

## 음성기술 (STT/TTS)

| 정식 용어 | 약어 | 교육자료 정의 | 고객 실제 표현 | 비고 |
|-----------|------|-------------|---------------|------|
| Speech-to-Text | STT | 음성을 텍스트로 변환하는 기술 | "STT", "음성인식", "텍스트 전환" | 상담 어드바이저의 핵심 기반 기술 |
| Text-to-Speech | TTS | 텍스트를 음성으로 변환하는 기술 | "TTS", "음성합성" | 콜봇 응답에 사용 |
| Neural TTS | 뉴럴 TTS | 딥러닝 기반 자연스러운 음성합성 | "뉴럴 TTS" | GPU 대신 CPU 코어 다수 사용. NH미팅에서 코어수(24~32) 논의 |
| Text Summary | TS | 텍스트 요약 기술 | "요약", "TS" | 에스원 미팅: "STTTS가 아니라 TS(Text Summary)" |
| Character Recognition Rate | CRR | STT 인식 정확도 측정 지표 | "인식률", "정확도" | 대신증권 시연: "EC STT 97% 인식 정확도" |
| 화자 분리 | - | 상담사/고객 음성을 스테레오로 분리 녹취 | "화자 분리", "RX/TX 녹취" | AICC 전제조건. 모노면 AI가 화자 구분 불가 |
| RTP (Real-time Transport Protocol) | RTP | 실시간 음성 전송 프로토콜 | "RTP", "음성 패킷" | SBC/VGW에서 추출하여 녹취/AI 콜봇에 전달 |
| SIPREC | - | SIP 기반 녹음 프로토콜. RTP 스트리밍 방식 | "SIPREC", "심맥" | SBC에서 AI 콜봇으로 RTP 전달 시 사용 |
| Voice-to-Voice | V2V | 음성 입력을 직접 음성으로 출력하는 모델 (STT/TTS 불필요) | "보이스 투 보이스", "보이스 모델" | 대신증권: "온프레미스 모델 아직 없음". 향후 기술 |
| Speech-to-Speech | S2S | V2V와 유사. 음성 직접 처리 모델 | "스피치 투 스피치" | NH미팅에서 리얼타임 API와 연관하여 논의 |
| MOS (Mean Opinion Score) | MOS | 음성 품질 평가 점수 (1~5점) | (직접 사용 안 함) | 클라우드 인터넷망 사용 시 MOS 0.78 수준 이슈 |
| EC STT/TTS | - | LG유플러스 자체 STT/TTS 엔진 (익시젠 기반) | "EC STT", "EC TTS", "익시 STT" | 대신증권 시연에서 97% 인식 정확도 시연 |
| WER (Word Error Rate) | WER | 단어 오류율. STT 인식 품질 측정 지표 (낮을수록 우수) | "WER", "더블유이알" | KB FCC STT PoC |
| CER (Character Error Rate) | CER | 문자 오류율. 한국어 STT 평가 시 WER보다 더 세밀한 지표 | "CER" | KB FCC STT, 쿠팡 STT |
| 실시간 파셜(Partial) STT | - | 음성이 끝나기 전에 중간 인식 결과를 실시간으로 화면에 표시하는 기능 | "파셜", "파셜 STT" | KB FCC STT 미팅 |
| LM 빌더 / LM 학습 도구 | - | STT의 언어모델(Language Model)을 도메인 데이터로 재학습시키는 관리 도구 | "LM 빌더" | KB FCC STT 미팅 |
| 핫워드(Hotword) | - | 특정 단어를 즉시 STT 인식 어휘에 반영하는 빠른 사전 등록 기능 | "핫워드" | KB FCC STT 미팅 |
| 스트리밍 모델 | - | 음성이 들어오는 즉시 인식을 처리하는 STT 엔진 모드 | "스트리밍", "스트리밍 모델" | KB FCC STT 미팅 |
| 컨텍스트 바이어스 (Context Bias) | - | STT 모델에 특정 키워드에 가중치를 부여해 인식 정확도를 높이는 기법 | "컨텍스트 바이어스" | 쿠팡 STT 미팅 |
| 스트리밍 TTS | - | 텍스트를 실시간 스트리밍 방식으로 음성 합성해 지연 없이 재생하는 TTS 방식 | "스트리밍 TTS" | CNS 미팅 |

## AI/LLM

| 멀티턴 (Multi-turn) | - | 여러 차례 대화를 주고받는 대화 방식 | "멀티턴", "문맥 유지", "이전 대화 보고 답변" | 우리캐피탈: "멀티턴 기능 되나요?" |
