# IPCC 구축 실무 교육자료
## 동양생명 구축 사례를 통한 IPCC 핵심 구성요소 이해

## 1. 교육 목적 & 전체 아젠다

### 교육 목적
- IPCC(Internet Protocol Contact Center)의 전체 구조를 실무 관점에서 이해
- 각 구성요소(PBX, CTI, IVR, 녹취)의 역할과 상호 연관성 파악
- 실제 제안서와 구축 사례를 통해 현업 적용 능력 배양

### 전체 아젠다
1. IPCC 전체 구조와 기존 콜센터와의 차이점
2. 핵심 구성요소별 상세 설명
3. 콜 흐름(Call Flow) 실습적 이해
4. 라이센스 체계 및 비용 구조

## 2. IPCC 전체 구조 (Big Picture)

### IPCC란?
- **Internet Protocol Contact Center**의 약자
- 기존 TDM 방식의 콜센터를 IP 기반으로 전환한 차세대 컨택센터
- 음성, 데이터, 애플리케이션을 통합하여 효율적인 고객 응대 가능

### 기존 콜센터(IPT)와 IPCC의 차이

| 구분 | IPT (Internet Protocol Telephony) | IPCC |
|------|-----------------------------------|------|
| 구성 | PBX + IVR (멘트 플레이 수준) | PBX + IVR + CTI + 녹취 + 상담AP |
| 기능 | 단순 전화 연결 및 안내 | 고객정보 연동, 라우팅, 통계, 녹취 |
| 데이터 | 통신 데이터만 처리 | 기관계 연동으로 고객 정보 활용 |

### 전체 아키텍처 흐름
```
고객 전화 → 통신사(PSTN) → E1/PRI 회선 → 보이스 게이트웨이
    → IP-PBX(교환기) → IVR → CTI → 상담원(Softphone + 상담AP)
    → CRM/기관계/녹취
```

## 3. 핵심 구성요소별 상세 설명

### 3.1 회선 및 프로토콜 변환

#### E1/PRI 회선
- **E1**: 유럽 표준 전송 방식, 2.048Mbps 대역폭, 32채널 중 30채널 실사용
- **PRI (Primary Rate Interface)**: ISDN 기반의 기업용 전화 회선
- 회선 계산 예시: "20회선 E1 사용" = 30채널 × 20 = 600채널 동시 통화 가능

#### 보이스 게이트웨이 (Voice Gateway)
- **역할**: TDM 방식(E1/PRI)을 IP 방식으로 프로토콜 변환
- **필요 이유**: IPCC는 IP 기반이므로 기존 전화망 신호를 IP로 변환 필수
- 시스코 등 네트워크 장비 업체가 자체 보이스 게이트웨이 보유

---

### 3.2 PBX (Private Branch Exchange) - 교환기

#### 정의
- 기업 내부의 전화 통신을 관리하는 핵심 장비
- 물리적 또는 논리적(소프트 스위치) 형태로 존재

#### PBX가 보유한 자원
| 자원 유형 | 설명 |
|-----------|------|
| 대표번호 | 1588-XXXX, 1577-XXXX 등 고객 인입 번호 |
| 회선번호 | 02-XXXX-XXXX 등 DID(Direct Inward Dialing) 번호 대역 |
| 내선번호 | 상담석별 고유 번호 (예: 1001, 1002, 1003...) |
| IVR 포트 | IVR 연결을 위한 논리적 내선 (예: 2001~2100) |
| 스킬 그룹 | 상담 그룹별 라우팅을 위한 그룹핑 정보 |

#### 다이얼링 플랜
- 전화 인입 시 어떻게 처리할지에 대한 시나리오 정의
- 예: 대표번호 → IVR 연결 → 서비스별 상담그룹 분배

---

### 3.3 CTI (Computer Telephony Integration) - 미들웨어

#### 정의
CTI는 IPCC의 **신경망**이자 **두뇌** 역할을 하는 미들웨어로, 교환기와 컴퓨터 시스템을 통합하여 통신과 데이터를 연계합니다.

#### CTI의 핵심 기능

| 기능 | 상세 설명 |
|------|-----------|
| **레지스터(Register)** | 교환기의 모든 내선/자원을 등록하여 상시 모니터링 |
| **이벤트 감시** | 콜 인입, 응답, 종료 등 모든 통화 이벤트를 실시간 추적 |
| **데이터 생성/정제** | 통신 패킷을 프로그래밍 언어로 번역하여 데이터화 |
| **로그 관리** | 모든 통화 이력을 로그로 기록 (장애 추적용) |
| **통계 생성** | 콜 통계, 상담사 퍼포먼스, 큐 통계 등 생성 |
| **라우팅 지시** | IVR에서 받은 정보로 상담석 연결 지시 |

#### 레지스터 vs 커넥션의 차이

| 구분 | 레지스터 (Register) | 커넥션 (Connection) |
|------|---------------------|---------------------|
| 방식 | 상시 모니터링, 상태값 추적 | 요청-응답 방식 통신 |
| 목적 | 이벤트 감지 및 로그 기록 | 데이터 조회 및 전송 |
| 대상 | PBX 내선, 상담석 | IVR, 기관계 시스템 |

#### CTI와 다른 시스템의 연결 관계
```
           ┌─────────────┐
           │   PBX       │ ←── 레지스터 (내선 모니터링)
           └─────────────┘
                 ↑
           ┌─────────────┐
           │    CTI      │ ←── 중앙 허브 역할
           └─────────────┘
           ↙    ↓    ↘
        IVR   상담AP   녹취
      (커넥션) (API)  (이벤트)
```

---

### 3.4 IVR (Interactive Voice Response)

#### ARS vs IVR의 차이
- **ARS**: 단순 음성 안내 (일방향)
- **IVR**: 고객과 **인터랙티브(양방향)** 통신, 데이터 연동 가능

#### IVR의 주요 기능

| 기능 | 설명 |
|------|------|
| 멘트 플레이 | "안녕하십니까, LG유플러스입니다" 등 안내 음성 재생 |
| 시나리오 분기 | "일반문의 1번, 결제문의 2번" 등 DTMF 입력에 따른 분기 |
| 기관계 연동 | 고객 정보(신용등급, 블랙리스트 등) 조회 |
| 전문 통신 | TCP/IP 기반 전문 통신으로 데이터 송수신 |

#### 기관계 연동 구조
```
IVR ←→ EAI/MCI (연계서버) ←→ 기관계 (핵심 시스템)
```
- **기관계**: 고객의 핵심 정보를 보유한 가장 중요한 시스템
- **보안상 직접 연결 불가**: EAI/MCI 등 연계 서버를 통해 통신
- 자바 전문(TCP/IP) 방식으로 데이터 교환

---

### 3.5 상담원 환경 (Softphone + 상담AP)

#### Softphone
- PC에 설치되는 소프트웨어 전화기
- CTI와 연동되어 전화 제어 기능 제공 (수신, 발신, 전환, 보류 등)
- 과거에는 OCX 모듈 방식으로 상담AP에 탑재

#### 스크린 팝업 (Screen Pop)
CTI가 정제한 고객 정보를 상담AP 화면에 자동 표시하는 기능

**데이터 흐름:**
1. CTI가 IVR로부터 고객 정보 수신 (전문 형태)
2. 딜리미터(구분자)로 데이터 파싱
3. 상담AP와 약속된 형식으로 전달
4. 화면에 고객명, 등급, 이력 등 팝업 표시

---

### 3.6 녹취 시스템

#### 녹취 시스템의 역할
- 상담 통화 내용을 음성 파일로 저장
- 품질관리(QA), 분쟁 해결, 법적 근거 확보 목적

#### CTI와 녹취의 연동
녹취 시스템은 단독으로 녹취 시작/종료를 판단할 수 없음

| 연동 항목 | 설명 |
|-----------|------|
| 녹취 시작 신호 | CTI의 콜 시작 이벤트 수신 시 녹취 시작 |
| 녹취 종료 신호 | CTI의 콜 종료 이벤트 수신 시 녹취 종료 |
| 메타데이터 매핑 | 전화번호, 타임스탬프, 인바운드/아웃바운드 구분 등 |
| 화자 분리 | 스테레오 녹취로 고객/상담사 음성 분리 |

#### 미러 포트 방식
- 네트워크 스위치에 미러 포트를 설정하여 음성 패킷 복제
- 복제된 패킷을 녹취 서버로 전송하여 저장

## 4. 실제 상담 흐름 시나리오

### 콜 인입부터 상담 종료까지

```
[1] 고객 전화 발신
    └→ 통신사(LG유플러스) PSTN 망

[2] 회선 인입
    └→ E1/PRI 회선으로 고객사 진입
    └→ 보이스 게이트웨이에서 IP 변환

[3] PBX 처리
    └→ 대표번호(1588-XXXX) 인식
    └→ CTI에 콜 이벤트 전달 (CID: 010-XXXX-XXXX)

[4] CTI 데이터 생성
    └→ 유니크 ID(UID) 생성 (16자리)
    └→ 타임스탬프 기록
    └→ IVR로 정보 전달

[5] IVR 처리
    └→ 안내 멘트 재생
    └→ DTMF 입력 수신 (예: 1번 일반문의)
    └→ 기관계 조회 (전화번호 → 고객명: 이광수, 등급: VIP)
    └→ 조회 결과를 전문으로 CTI에 반환

[6] CTI 라우팅
    └→ 서비스 유형에 따른 스킬그룹 결정
    └→ PBX에 상담석 연결 지시
    └→ 해당 내선(1001번)으로 호 전달

[7] 상담원 응대
    └→ Softphone 벨 울림
    └→ 스크린 팝업 (고객명, 등급, 상담이력 표시)
    └→ 전화 수신 → 상담 진행

[8] 녹취 연동
    └→ CTI 콜 시작 이벤트 → 녹취 시작
    └→ CTI 콜 종료 이벤트 → 녹취 종료
    └→ 메타데이터 매핑 후 저장

[9] 통계 및 모니터링
    └→ CTI에서 실시간 대시보드 제공
    └→ 관리자가 콜 현황, 상담사 상태 모니터링
```

## 5. 장비 구성 및 라이센스 체계

### 5.1 어플라이언스 vs 범용 서버

| 구분 | 어플라이언스 장비 | 범용 서버 |
|------|-------------------|-----------|
| 정의 | 벤더 전용 하드웨어+소프트웨어 일체형 | HP, Dell 등 일반 서버에 SW 설치 |
| 사용 벤더 | 어바이어, 시스코 | 브리지텍, 넥서스, 제네시스 |
| 유지보수 | 벤더사만 가능 | HW/SW 분리 유지보수 가능 |
| 보안 이슈 | 루트 계정 사용으로 보안 취약 가능 | 하위 권한 설정 가능 |

### 5.2 가상화 구성
- VMware 등 가상화 플랫폼 위에 IPCC 솔루션 구성
- 관리 효율성 향상: 단일 콘솔에서 전체 시스템 모니터링/관리
- 장비 축소: 물리적 박스 수 감소

### 5.3 라이센스 종류

| 라이센스 유형 | 설명 |
|---------------|------|
| **Named License** | 특정 사용자/자리에 고정 할당 (TO 기준) |
| **Concurrent License** | 동시 접속자 기준 (실제 사용자 수) |
| **Perpetual License** | 영구 라이센스 (1회 구매) |
| **Subscription License** | 구독형 라이센스 (월/년 단위 과금) |

### 5.4 라이센스 트렌드 변화
- **과거**: 영구 라이센스(Perpetual) 중심
- **현재**: 구독형(Subscription) 라이센스가 주류
  - 2023년 이후 글로벌 벤더들 대부분 구독형 전환
  - 클라우드 전환과 함께 가속화
- **국내 상황**: 고객은 5년 약정 선호, 글로벌은 3년 기준

### 5.5 비용 구조
| 항목 | 내용 |
|------|------|
| PBX 라이센스 | 내선 수 기준 |
| CTI 라이센스 | 모니터링 대상 자원 수 기준 |
| IVR 라이센스 | 포트 수 기준 |
| 녹취 라이센스 | 녹취 채널 수 기준 |
| 서버 비용 | 하드웨어 + 가상화(VMware) |

## 6. 교육 중 강조된 포인트 요약

### CTI의 중요성
> "CTI가 없으면 IPCC가 아니다. CTI가 있어야 콜센터의 모든 기능이 완성된다."

- CTI는 모든 시스템의 허브 역할
- 장애 발생 시 CTI 엔지니어가 로그 분석으로 원인 추적
- 통계, 모니터링, 라우팅 모두 CTI 기반

### 기관계 연동의 특수성
- 기관계는 가장 보안이 철저한 시스템
- 직접 연결 불가, 반드시 EAI/MCI 등 연계 서버 경유
- 금융권의 경우 수십 년간 축적된 전문 인터페이스 존재

### 벤더별 특성
| 벤더 | 특징 |
|------|------|
| 시스코(C) | 네트워크 강점, 자체 보이스 게이트웨이 |
| 어바이어(A) | 어플라이언스 장비, 전통적 강자 |
| 제네시스(G) | 클라우드 전환 선도 |
| 브리지텍(BT) | IVR 특화, 금융권 점유율 높음 |

### 음질 관련
- 과거: TDM vs IP 음질 논쟁 존재
- 현재: IP 방식 음질 충분히 검증됨, 더 이상 이슈 아님
- 클라우드: 인터넷망 사용 시 약간의 품질 저하 가능 (MOS 0.78 수준)
  - 다이렉트 커넥트(전용선) 연결로 해결

## 7. 한 페이지 요약 (Summary)

### IPCC 핵심 구성도
```
┌────────────────────────────────────────────────────────────┐
│                        IPCC 구성도                          │
├────────────────────────────────────────────────────────────┤
│                                                            │
│  [고객] → [PSTN] → [E1/PRI] → [VoIP GW] → [IP-PBX]        │
│                                              ↓             │
│                                           [CTI] ←──────┐   │
│                                          ↙  ↓  ↘       │   │
│                                       [IVR][AP][녹취]  │   │
│                                         ↓              │   │
│                                    [기관계/CRM] ───────┘   │
│                                                            │
└────────────────────────────────────────────────────────────┘
```

### 핵심 요약 5가지

1. **IPCC = PBX + CTI + IVR + 녹취 + 상담AP**
   - IPT(단순 전화)와 달리 고객 정보 연동 및 지능형 라우팅 가능

2. **CTI가 핵심 (미들웨어)**
   - 모든 시스템을 연결하는 허브
   - 레지스터(모니터링) + 데이터 정제 + 통계 + 라우팅 제어

3. **IVR은 기관계와 통신**
   - 고객 정보 조회 및 시나리오 분기
   - EAI/MCI를 통한 보안 통신

4. **라이센스는 석(席) 기준**
   - PBX, CTI, IVR, 녹취 각각 라이센스 필요
   - 현재 트렌드는 구독형(Subscription)

5. **벤더별 제품 이해 필요**
