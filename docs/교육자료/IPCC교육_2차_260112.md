# IPCC 교육 요약 (2026.01.12)

## 1. IPCC 기본 구성 복습

### 핵심 구성요소
1. **PBX** (교환기)
2. **CTI** (미들웨어)
3. **IVR** (음성응답)
4. **녹취**
5. **상담 어플리케이션**

### 전체 흐름
```
고객 → 통신사(PSTN) → E1/PRI → 보이스 게이트웨이 → PBX
    → CTI → IVR → 상담석(Softphone + 상담AP) → 녹취 → 통계
```

---

## 2. PBX (교환기) 자원

| 자원 | 설명 | 예시 |
|------|------|------|
| 대표번호 (Q) | 고객 인입 번호 | 1588-XXXX |
| 내선번호 | 상담석 할당 번호 | 1001~1100 (5자리) |
| IVR 포트 | IVR 연결용 내선 | 2001~2100 |
| 스킬 그룹 | 상담 그룹 라우팅 | - |

### Q(큐) 용어 정리
- **어바이어**: VDN
- **시스코**: CDN
- **제네시스**: 라우팅 포인트

---

## 3. CTI의 역할

> "CTI가 없으면 IPCC가 아니다"

### 핵심 기능
- **레지스터**: PBX의 모든 DN 자원 모니터링
- **이벤트 감지**: 콜 인입/응답/종료 추적
- **데이터 생성**: 16진수 프로토콜 → 사람이 읽을 수 있는 데이터로 변환
- **라우팅 지시**: IVR → 상담석 연결 제어
- **통계 생성**: 호통계, 상담사 퍼포먼스, 큐 통계

### CTI 연결 구조
```
       [PBX] ←── 레지스터 (DN 모니터링)
         ↑
       [CTI] ←── 중앙 허브
      ↙  ↓  ↘
   [IVR][상담AP][녹취]
```

---

## 4. IVR 기능

- 멘트 플레이: "안녕하십니까, LG유플러스입니다..."
- 시나리오 분기: DTMF 입력에 따른 분기 (if문 구조)
- **기관계 연동**: EAI/MCI를 통한 고객 정보 조회
  - 과거: TCP/IP 소켓 통신
  - 현재: REST API (JSON 형식)

---

## 5. 녹취 시스템

### 연동 방식
- CTI 이벤트 수신 → 녹취 시작/종료 판단
- **미러 포트**: 스위치에서 RTP 패킷 복제
- 데이터 매핑: CTI 데이터 + PBX 음성 패킷 결합

---

## 6. 통계 체계

### 통계 종류
| 구분 | 설명 |
|------|------|
| 호통계 | 콜 + 디지털 채널 전체 통계 |
| 상담사 통계 | 퍼포먼스, 상태값 |
| 시간별 통계 | 15분/30분/일/주/월/연 |
| 상담 이력 | 실제 상담 내용 기반 (더 정확) |

### 통합 통계
- 인프라 통계 + 상담 이력 + 녹취 연동
- 통계 클릭 시 음성 청취 가능

## 7. PDS (Predictive Dialing System)

### 정의
자동 아웃바운드 발신 시스템

### 사용 사례
- 해피콜, 서베이, 여론조사
- 공지/알림, 혜택 정보 안내

### 3가지 모드

| 모드 | 설명 | 특징 |
|------|------|------|
| **Preview** | 상담사가 직접 클릭 발신 | 수동, 민감 고객용 |
| **Progressive** | 가용 자원 수만큼 발신 | 안정적 |
| **Predictive** | 가용 자원 이상 발신 (예: 120%) | 알고리즘 예측, DB 소진 빠름 |

### PDS 구성요소
1. **콜링 리스트**: 발신 대상 DB
2. **캠페인**: 모드/룰 설정 (리트라이 조건 등)
3. **관리 모듈**: 실행/모니터링/통계

### 연결 방식
- **CTI 사이드**: CTI가 모니터링
- **PBX 사이드**: PDS가 직접 DN 모니터링

### 국내 시장
- 주요 업체: **샌드박스** (독과점)
- 테라, 한맥 등 대부분 철수

## 8. 옴니채널 vs 멀티채널

| 구분 | 멀티채널 | 옴니채널 |
|------|----------|----------|
| 정의 | 채널 분리 운영 | 채널 통합 연동 |
| 데이터 | 개별 관리 | 유기적 연동 |
| 예시 | 전화(시스코) + 채팅(채널톡) 분리 | 전화+채팅+SNS+키오스크 통합 |

### 용어 정리
- **콜(Call)**: 보이스만
- **호(呼)**: 보이스 + 디지털 채널 전체
- **인터랙션**: 호와 동일 (글로벌 용어)

### 옴니채널 현실
- 완전한 구현 사례 거의 없음
- 이유: 웹 기반 한계, 데이터 용량 급증, 기능 제약
- 제안서는 옴니채널로 포장, 실제 구축은 제한적

## 9. SBC와 미디어 게이트웨이

### 벤더별 구성

| 벤더 | 프로토콜 변환 장비 | 교환기 |
|------|-------------------|--------|
| 어바이어, 시스코 | 보이스 게이트웨이 (G450) | 전통적 PBX |
| 브리지텍, EICN | **SBC** | 소프트 스위치/SIP 스위치 |

### 미디어 게이트웨이 vs SBC

| 구분 | 미디어 게이트웨이 | SBC |
|------|-------------------|-----|
| 변환 | TDM → IP | IP → IP (+E1 카드로 TDM 수용) |
| 기능 | 단순 변환 | 라우팅, 이중화, RTP 스트리밍 |

### SBC 제품
- **Oracle SBC**: EICN에서 주로 사용
- **AudioCodes**: 브리지텍에서 사용, 가격 저렴 (이스라엘 제품)

### AI 콜봇 연동
```
[SBC] ──┬── [소프트 스위치] → [상담AP] → [녹취]
        │
        └── [AI 콜봇] ← RTP 스트리밍 (SIPREC/심맥)
              ↓
           [STT/TTS]
```
- SBC: RTP 패킷 2개까지 스트리밍 가능
- 어바이어/시스코: 보이스 게이트웨이에서 RTP 추출
- 브리지텍/EICN: SBC에서 RTP 추출

## 10. 원벤더 솔루션 트렌드

### 원벤더 vs 멀티벤더

| 구분 | 원벤더 | 멀티벤더 |
|------|--------|----------|
| 정의 | 한 업체가 전체 제공 | 영역별 다른 업체 조합 |
| 적용 | 중소형, 그룹사 | 대형 금융사 |

### 주요 벤더 현황

| 벤더 | 특징 |
|------|------|
| **브리지텍** | 막강, 농협 수주, 금융권 강세 |
| **EICN** | 가성비 좋음, 소형 사이트, 상주 지원 제한 |
| **어바이어** | 불확실한 미래 |
| **시스코** | 안정적, 특정 대형 고객 위주 |
| **제네시스** | 클라우드 전환 선도 |
