## 1. 해결해야 할 문제

### 팀 현황
- **AI사업2팀**: AICC 구축형 사업 리딩 조직 (고객회의·사업지원이 주 역할)
- **10명** 중 IPCC/AICC 도메인 지식 보유자 **2명**, 나머지 **8명**은 신규전입·경력직으로 도메인 학습 필요
- **1~2년**: IPCC/AICC 도메인 적응 소요 기간

### 네 가지 핵심 문제

#### 1. 도메인 지식 편중
10명 중 2명만 IPCC/AICC 도메인 지식 보유 → 8명이 고객회의·사업지원 현장에서 즉시 기여 어려움

#### 2. 신뢰 손실 기간
AICC 구축 사업 특성상 고객(컨택센터 운영자, IT담당자)은 IPCC/AICC 전문성을 기대. 도메인 적응 기간 동안 고객 접점에서 신뢰 손실 발생

#### 3. 지식과 적용의 괴리 (핵심 갭)
AICC/IPCC 상품 교육 자료는 있으나, 현장에서 고객이 쓰는 컨택센터 용어(ACD, IVR, CTI, WFM 등)를 해독하고 구축 상황별 판단을 내리는 능력은 교실에서 못 배움

#### 4. 도제식 모델의 한계
도메인 전문가 2명이 8명을 동시에 코칭하는 것은 물리적으로 불가. 고객회의·사업지원 일정이 빡빡한 상황에서 OJT 여력 부족

### 현장 인사이트
"경력직으로 왔는데 AICC 도메인은 처음이라 미팅 가면 용어를 못 알아들어요. IPCC 아키텍처도 모르고, 고객이 물어보는 ACD 라우팅이나 IVR 시나리오 얘기에 끼지를 못해요. 최소 1~2년은 걸린다고 하더라고요"

### 두 가지 핵심 병목

#### 기존 고객 인수 시 신뢰 문제
신규전입자가 베테랑 대신 담당하면, AICC 구축 고객이 IPCC/AICC 역량 부족을 즉시 감지 → 신뢰 상실

#### 신규 고객 기회 발굴
AICC 도입을 검토하는 고객과 첫 대화를 여는 것 자체가 안 됨. IPCC 현황 파악부터 AICC 전환 가치 제안까지, 도메인 지식 없이는 "입떼기"가 불가능

## 2. 솔루션 & 핵심 설계 원칙

### 비전
키엔스의 매일 롤플레이 문화에서 영감. AI를 통해 시간/장소/상대방 제약 없이 **IPCC/AICC 도메인 반복 연습 환경**을 제공하여, 신규전입·경력직 담당자가 몇 년 걸리는 도메인 경험치를 압축.

### 원칙 1: 평가 도구가 아닌 개인 연습 도구

사용 이력/점수/결과는 **본인만** 열람. 리더/관리자에게 개인별 데이터 미제공. 인사 평가 반영 절대 불가. "영어 연습 앱처럼 혼자 쓰는 도구."

> "콜센터 QA처럼 되면 진짜 아무도 안 할 거예요. 점수가 공개되면 누가 시뮬레이션 해보겠어요"

### 원칙 2: RAG 기반 차별화

LG U+ AICC/IPCC 상품 데이터, 구축 사례, 내부 사업팀 노하우를 RAG로 구축. ChatGPT/Gemini에 물어보는 것과 **차원이 다른 정확도**.

> "제미나이나 GPT에 물어보고 가는 거랑 차이가 있을까. RAG 기반이 아니면 크게 차별화가 쉽지 않을 것 같은데"

### 원칙 3: 데이터 프라이버시 보호

학습 데이터는 **비식별화된 요약본 기반**으로 수집. 개인 식별 불가 처리. 업무 관련 데이터만 활용하며, 개인정보 보호 원칙 준수.

## 3. 단계별 구현 전략

### Stage 1: IPCC/AICC 도메인 학습 + 롤플레이 (MVP) ✅ 구현 완료
**일정**: 3-4월 (당초 4-5월 → 앞당김)

IPCC/AICC 도메인 지식 학습과 고객 미팅 롤플레이를 통합한 MVP. RAG 기반 지식 검색, 퀴즈, 용어 사전, 아키텍처 가이드, AI 고객 롤플레이 시뮬레이션, 피드백을 하나의 도구에 제공.

> "IPCC 아키텍처나 AICC 기능 Q&A처럼 정리해서 고객 미팅 가면 80%는 대응할 수 있어. 자신감이 다르지"

**구현된 기능 (4개 탭)**

| 탭 | 기능 | 상태 |
|----|------|------|
| 🎭 롤플레이 | AI 고객과 사업 미팅 시뮬레이션 (5개 시나리오, 3단계 난이도) | ✅ |
| 🎯 도메인 퀴즈 | 8개 카테고리 × 3단계 난이도 퀴즈 (사전 생성 풀 즉시 출제) | ✅ |
| 📖 용어 사전 | 컨택센터 용어 검색 (정식 용어 ↔ 고객 현장 표현 매핑) | ✅ |
| 🏗️ 아키텍처 가이드 | IPCC 구조, 콜 플로우, AICC 전환, 벤더 비교 | ✅ |

**롤플레이 시나리오 (실제 고객 미팅 기반)**

| 시나리오 | 기반 | 고객 역할 | 난이도 |
|----------|------|-----------|--------|
| 솔루션 시연 + 사업 효과 설명 | 대신증권 미팅 | 디지털전략팀 과장 | ★★★★ |
| 기술 견적 협의 | NH투자증권 미팅 | IT기획부 차장 | ★★★★★ |
| 보이스봇 요건정의 / RFP 지원 | 에스원 미팅 | 고객서비스팀 대리 | ★★★ |
| KMS/RAG 상품 소개 Q&A | 우리금융캐피탈 미팅 | 디지털혁신팀 과장 | ★★★ |
| AICC 도입 초기 미팅 (탐색) | 한화손해보험 미팅 | 고객서비스부 팀장 | ★★★★ |

**고객 페르소나 (3단계)**

| 난이도 | 스타일 |
|--------|--------|
| 초급 | 친근한 고객 — 협조적, 질문 위주 |
| 중급 | 꼼꼼한 실무자 — 수치 요구, 경쟁사 비교, 재질문 |
| 고급 | 공격적 의사결정자 — ROI 추궁, 가격 압박, 시간 압박 |

**퀴즈 카테고리 (8개)**
- IPCC 아키텍처 (PBX, CTI, IVR, 녹취)
- 음성기술 (STT, TTS, 화자분리)
- AI/LLM (RAG, 콜봇, 챗봇)
- 상담운영 (ACD, WFM, QA)
- 도입/구축 (PoC, 레퍼런스, 연동)
- 비용/라이센스
- AICC 기능 (상담 어드바이저, 보이스봇)
- 컨택센터 용어 (현장 표현 vs 정식 용어)

### Stage 2: 고도화 + 사후 복기
**일정**: 5-7월

MVP를 기반으로 롤플레이 시나리오 확장, 피드백 정교화, 사후 복기 기능 추가.

**주요 기능**
- 시나리오 라이브러리 확장 (IPCC→AICC 전환, 신규 구축, 기능 확장 등)
- 도메인 전문가 3명의 BP 기반 모범 답변 고도화
- 미팅 복기 분석 (고객회의 후 메모 입력 → "고객이 진짜 원한 것" 해석)
- 내부 커뮤니케이션 초안 생성
- 액션 투두 정리
- 개인 성장 추적 (학습 이력, 약점 분석)

---

### Stage 3: 실전 어드바이저
**일정**: 7-9월

AICC 구축 고객회의 중 태블릿/모바일로 IPCC/AICC 상품·고객 구축 정보 즉시 조회. 연습 도구를 넘어 사업지원 실전 무기로.

> "고객이 IVR 시나리오 물어보면 태블릿에서 바로 찾아서 보여주는 거지. 연습용도 좋지만 현장 무기가 딱 돼가지고"

**주요 기능**
- 실시간 IPCC/AICC 상품·구축 사례 조회
- AICC 구축 상황별 대응 가이드
- STT 연동 (탐색)

## 4. 기술 아키텍처 (현재 구현)

### 시스템 구성

```
[사용자] → [Streamlit Web UI] → [Claude CLI] → [Claude Pro/Max 구독]
                                ↕
                         [RAG Pipeline]
                         ├── ChromaDB (벡터 저장소)
                         └── HuggingFace Embeddings (로컬)
```

### 기술 스택

| 구성요소 | 기술 | 비고 |
|----------|------|------|
| **LLM** | Claude CLI (Pro/Max 구독) | API Key 불필요, 무료 |
| **임베딩** | intfloat/multilingual-e5-small | 로컬 실행, API 불필요 |
| **벡터 DB** | ChromaDB | 로컬 파일 기반 |
| **프레임워크** | LangChain + Streamlit | Python 3.9+ |
| **데이터** | 12개 문서 → 312 chunks | 교육자료 + 고객미팅 기록 |

### 파일 구조

```
081_Simulation/
├── app.py                  # Streamlit 메인 앱 (4개 탭)
├── config.py               # 프로젝트 설정
├── llm.py                  # Claude CLI 래퍼
├── rag.py                  # RAG 파이프라인 (검색 + 답변)
├── ingest.py               # 문서 임베딩 + 벡터 DB 구축
├── roleplay.py             # 롤플레이 엔진 (시나리오/페르소나)
├── feedback.py             # 롤플레이 피드백 분석
├── quiz.py                 # 도메인 퀴즈 (풀 기반 즉시 출제)
├── generate_quiz_pool.py   # 퀴즈 풀 사전 생성 스크립트
├── quiz_pool.json          # 사전 생성된 퀴즈 (즉시 로딩)
├── chroma_db/              # ChromaDB 벡터 저장소
├── docs/                   # 학습 데이터
│   ├── 교육자료/           # AICC/IPCC 상품 교육 문서
│   ├── 고객미팅/           # 실제 고객 미팅 기록
│   ├── 도메인_용어사전.md
│   ├── 고객질문_패턴DB.md
│   └── 롤플레이_시나리오.md
└── requirements.txt
```

### 핵심 설계 결정

| 결정 | 이유 |
|------|------|
| Claude CLI (API 아님) | Pro/Max 구독으로 무료 사용, API 키 관리 불필요 |
| 로컬 임베딩 | API 비용 제로, 오프라인 가능 |
| 퀴즈 풀 사전 생성 | CLI 호출 대기 시간(~15초) 제거, 즉시 출제 |
| ChromaDB | 경량 벡터 DB, 설치 간편, 로컬 파일 기반 |

## 9. 사업화 확장 가능성

### 음성 AI의 전략적 중요성

**홍범식 CEO, MWC 2026 기조연설** (바르셀로나):
> "음성은 사람과 AI를 연결하는 가장 자연스러운 인터페이스. 화자를 식별하고 질문의 의도, 맥락과 감정까지 파악하는 Voice AI가 미래 통신의 핵심이다."
> "데이터만 넘기면 20년 전 전철 밟게 된다. 통신사가 가진 고유한 보이스 데이터가 차별화 자산이다."

### 익시오(ixi-O) 전략과의 정합성

홍범식 사장이 MWC 기조연설에서 제시한 핵심: **음성 데이터 기반 AI가 LG U+의 미래**. 사업팀 AI 롤플레이 에이전트는 B2B 고객 현장의 고유한 보이스 데이터를 축적하는 프로젝트로, 익시오의 B2B 확장 기반이 될 수 있다.

### 보이스 데이터의 전략적 가치

사업팀 고객 대화 데이터는 화자 식별, 질문 의도, 맥락, 감정 분석의 최적 학습 자원. 이 데이터는 경쟁사가 가질 수 없는 **LG U+만의 고유 자산**이 된다.

### 글로벌 시장 검증

- Hyperbound, Mindtickle 등 해외 유사 솔루션 다수 존재
- **시장규모**: $62B+ AI 코칭 시장
- **성장률**: 28.3% CAGR
- **한국 현황**: B2B 특화 솔루션은 사실상 공백

### AI 중심 SW 기업으로의 전환

홍범식 사장 비전 "통신을 넘어 글로벌 AI SW 기업으로 도약". 사업팀 현장 AI 도구는 이 전환의 실증 사례로, B2B Enterprise AI Full-Stack 전략의 한 축이 될 수 있다.

현 단계에서는 내부 도구로 검증 우선. 익시오 연계 및 사업화는 Stage 2 이후 검토.
